### 42 接雨水
给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。
—— 计算蓝色部分面积
![[others/Pasted image 20251109104645.png]]


- 当前节点，`蓝色面积= min(其左边最高，其右边最高)-当前节点高度`


1️⃣ 双指针法
- 下标 i 处能接的雨水量由 leftMax[i] 和 rightMax[i] 中的最小值决定
- 当两个指针没有相遇时
	- 使用 height[left] 和 height[right] 的值更新 leftMax 和 rightMax 的值；
	- 如果 height[left]<height[right]，则必有 leftMax<rightMax，下标 left 处能接的雨水量等于 leftMax−height[left]，将下标 left 处能接的雨水量加到能接的雨水总量，然后将 left 加 1（即向右移动一位）；
	- 如果 height[left]≥height[right]，则必有 leftMax≥rightMax，下标 right 处能接的雨水量等于 rightMax−height[right]，将下标 right 处能接的雨水量加到能接的雨水总量，然后将 right 减 1（即向左移动一位）。
- 当两个指针相遇，则遍历完整个数组，得到 water 值
```python
def trap(height: list[int]) -> int:
    if not height or len(height)<3:
        return 0
    
    # 双指针所需变量
    left,right = 0,len(height)-1
    left_max,right_max=0,0
    water = 0
    # 使用双指针向中间移动
    while left<right:
        left_max=max(left_max,height[left])
        right_max=max(right_max,height[right])

        # 根据较小一侧确定移动方向
        if left_max<right_max:
            water+=left_max-height[left]
            left+=1
        else:
            water+=right_max - height[right]
            right-=1
    return water
```

### 76  最小覆盖子串
【解题】
- [🔗 76](https://github.com/huzibo-yyds/LeetCode/commit/e0b031e1304893e0f8901f1ec19deeefd92d5a89)

【题目】
- 给你一个字符串 `s` 、一个字符串 `t` 。
-  返回 `s` 中涵盖 `t` 所有字符的最小子串。
- 如果 `s` 中不存在涵盖 `t` 所有字符的子串，则返回空字符串 `""` 。

【解题思路】***滑动窗口 | 哈希表***
1. 使用两个指针来维护一个窗口
2. 窗口扩张，直至当前窗口，覆盖字串t中所有字符
3. 窗口收缩，左指针右移，直到不符合覆盖条件
	1. 在指针右移的过程中，每次更新最小字串的信息
- 为了高效判断窗口覆盖t，使用哈希表记录字串中字符频率

```python
def minWindow(s: str, t: str) -> str:
    if not t or not s or len(s)<len(t):
        return ""

    # 使用hash表记录字符频率
    need={} # 1️⃣ 哈希表1，用来统计t中信息
    for char in t:
        need[char]=need.get(char,0) + 1
        # .get() # Return the value for key if key is in the dictionary, else default.

    # 滑动窗口相关变量
    left,right=0,0
    window = {} # 2️⃣ 哈希表2，统计当前窗口信息

    # 最小覆盖字串的起始位置与长度
    start = 0
    min_len=float('inf')

    # 记录窗口中已经满足need条件的字符种类数
    valid=0 

    while right < len(s):
        c=s[right]
        right+=1

        # 窗口内字符统计
        if c in need:
            window[c]=window.get(c,0)+1
            if window[c] == need[c]:
                valid+=1

        # 满足覆盖时，窗口收缩
        while valid==len(need):
            if right-left<min_len:
                start = left
                min_len = right - left
            
            d=s[left]
            left+=1

            if d in  need:
                if window[d]==need[d]:
                    valid-=1
                window[d]-=1
    
    return "" if min_len==float('inf') else s[start:start+min_len]
```

### 206 翻转链表
基础功

**迭代**实现
从前往后逐个遍历
- `prev`，旧链表的下一个元素
- `current`，当前元素（从前往后遍历）
	- `next_temp`，current 后一个元素
```python
def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
		"""
		迭代反转链表（时间 O(n)，空间 O(1)）
		最优解之一：必须遍历所有节点，因此时间下界是 O(n)。
		"""
		# prev 指向已经反转好的前半部分，current 指向待处理节点
		prev = None
		current = head
		
		while current:  # 📍需要记录三个节点，前一个，当前，后一个
				# 保存下一个节点
				next_temp = current.next
				# 反转当前节点的指针
				current.next = prev
				# 移动指针
				prev = current
				current = next_temp
		
		# prev 是新链表的头
		return prev
```

**递归**实现

```python
def reverseList_recursive(self, head: Optional[ListNode]) -> Optional[ListNode]:
		"""
		递归反转链表（时间 O(n)，空间 O(n) 递归栈）
		"""
		# 空链表或单节点时，直接返回
		if not head or not head.next:
				return head

		# 反转后半部分，new_head 是反转后的头
		new_head = self.reverseList_recursive(head.next)
		# 将当前节点接到反转后的尾部
		head.next.next = head
		head.next = None
		return new_head
```

### 📗 dummy + tail 模板
`dummy + tail` 是链表题里最实用的“**通用拼接模板**”。

核心定义：
- `dummy`：**虚拟头结点**（不存业务数据），作用是统一处理“第一个节点怎么接”的边界问题。
- `tail`：结果链表当前的尾指针，始终指向“已构建部分”的最后一个节点。

好处：
1. 不需要单独判断“结果链表是否为空”。
2. 不需要反复处理“头节点特殊情况”。
3. 逻辑稳定：每次都做同一件事——`tail.next = 某节点`，然后 `tail = tail.next`。

模板动作（4步）
1. 初始化：`dummy = ListNode()`，`tail = dummy`
2. 循环中选一个节点接到尾部：`tail.next = pick`
3. 尾指针后移：`tail = tail.next`
4. 收尾返回：`return dummy.next`（跳过虚拟头）

## 图论
### 图论核心概念&算法
#### 基本概念
- **图 (Graph)**：由**顶点 (Vertex)** 的集合和**边 (Edge)** 的集合组成。一条边连接两个顶点。
- **有向图 (Directed Graph)**：边有方向。如果有一条从顶点 A 到顶点 B 的边，你只能从 A 到 B，不能从 B 到 A（除非有另一条反向的边）。
- **无向图 (Undirected Graph)**：边没有方向。如果 A 和 B 之间有一条边，你可以从 A 到 B，也可以从 B 到 A。
- **权重 (Weight)**：可以给边赋予一个数值，称为权重。例如，在地图上，权重可以代表两个城市之间的距离或时间。有权重的图称为**加权图 (Weighted Graph)**。

#### 存储方式
主要有两种方法：
- **邻接矩阵 (Adjacency Matrix)**：
    - 用一个二维数组 `graph[i][j]` 来表示图。
    - `graph[i][j] = 1` (或权重值) 表示从顶点 `i` 到顶点 `j` 有一条边。
    - `graph[i][j] = 0` (或无穷大) 表示没有边。
    - **优点**：查询两个顶点之间是否有边非常快 (O(1))。
    - **缺点**：如果图很稀疏（边很少），会浪费大量空间 (O(V²)，V是顶点数)。


- **邻接表 (Adjacency List)**：
    - 用一个数组（或哈希表），数组的每个元素对应一个顶点，其值是一个列表（或链表），存储着所有与该顶点相邻的顶点。
    - `adj[i]` 包含所有 `i` 可以直接到达的顶点。
    - 这是**最常用**的表示方法。
    - **优点**：节省空间 (O(V+E)，E是边数)。
    - **缺点**：查询两个顶点之间是否有边需要遍历列表 (O(k)，k是度)。


#### 核心图遍历算法
这是*解决几乎所有图论问题的基础*。
- **广度优先搜索 (Breadth-First Search, BFS)**：
    - 从一个起始点开始，一层一层地向外扩展。先访问所有距离为1的邻居，再访问距离为2的邻居，以此类推。
    - **实现**：通常使用一个**队列 (Queue)**。
    - **应用**：求无权图的最短路径、层序遍历。
    
- **深度优先搜索 (Depth-First Search, DFS)**：
    - 从一个起始点开始，沿着一条路径走到头，然后再回溯，探索其他路径。
    - **实现**：通常使用**递归 (Recursion)** 或**栈 (Stack)**。
    - **应用**：判断图是否有环、寻找连通分量、拓扑排序。

#### 常见图论算法
- **最短路径算法**：
    - **Dijkstra 算法**：计算一个顶点到所有其他顶点的最短路径（不能有负权边）。
    - **Bellman-Ford 算法**：功能同上，但可以处理有负权边的情况。
    - **Floyd-Warshall 算法**：计算任意两个顶点之间的最短路径。

- **最小生成树 (Minimum Spanning Tree, MST)**：
    - 在一个加权无向图中，找到一棵连接所有顶点的树，使得树上所有边的权重之和最小。
    - **Prim 算法**
    - **Kruskal 算法**

- **拓扑排序 (Topological Sort)**：
    - 对一个**有向无环图 (DAG)** 的顶点进行排序，使得对于每一条有向边 (u, v)，u 都在 v 的前面。
- **并查集 (Union-Find / Disjoint Set Union, DSU)**：
    - 一种高效的数据结构，用于处理一些不相交集合的合并及查询问题。常用于判断图的连通性或是否存在环（例如在 Kruskal 算法中）。

### 什么情况使用BFS
BFS 可以看成是 *层序遍历*。从某个结点出发，BFS 首先遍历到距离为 1 的结点，然后是距离为 2、3、4…… 的结点。因此，BFS 可以用来求最短路径问题。BFS 先搜索到的结点，一定是距离最近的结点。

举例，如994 橘子腐烂问题
返回直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。翻译一下，实际上就是求**腐烂橘子到所有新鲜橘子的最短路径**。

❓如何写BFS
✅BFS需要使用==队列==，示意
```python
while queue 非空:
	node = queue.pop()
    for node 的所有相邻结点 m:
        if m 未访问过:
            queue.push(m)
```

如果需要**逐层处理**
```python
depth = 0 # 记录遍历到第几层
while queue 非空:
    depth++
    n = queue 中的元素个数
    循环 n 次:
        node = queue.pop()
        for node 的所有相邻结点 m:
            if m 未访问过:
                queue.push(m)
```


### 200 岛屿数量
【题意】
给你一个由 '1'（陆地）和 '0'（水）组成的的二维网格，请你*计算网格中岛屿的数量*。 (邻接矩阵保存图)
岛屿总是被水包围，并且每座岛屿只能由水平方向和/或竖直方向上相邻的陆地连接形成。
此外，你可以假设该网格的四条边均被水包围。


【code】

【方法】
核心思想，*寻找到‘1’，岛屿数+1，把当前属于该岛屿的节点都置0*
1. 遍历整个网格。
2. 当我们遇到一个 '1'（陆地）时，我们就找到了一个新岛屿的起点，此时岛屿数量加一。
3. 接着，我们需要把这个岛屿的所有部分都找到，并做一个标记（比如把 '1' 变成 '0'），以防止重复计数。这个过程就像把整个岛屿“*淹没*”掉。
4. 继续遍历网格，直到所有格子都被检查过。

【实现】
1、DFS
```python
class Solution:
    def numIslands(self, grid: list[list[str]]) -> int:
        if not grid:
            return 0
        rows,cols=len(grid),len(grid[0])
        num_islands = 0
        def dfs(r,c):
            '''深度优先搜索 核心思想：优先探索一个分支到底，再回溯探索其他分支
            '''
            # 1、边界检查 + 终止条件
            # 如果越界，或者是水，或者是已经访问过的陆地，就返回
            if not (0 <= r < rows and 0 <= c < cols and grid[r][c] == '1'):
                return
            # 2. 标记当前陆地为已访问（淹没）
            grid[r][c] = '0'
            # 3. 向四个方向递归探索
            dfs(r + 1, c)  # 下
            dfs(r - 1, c)  # 上
            dfs(r, c + 1)  # 右
            dfs(r, c - 1)  # 左
        # 📍主要执行过程
        for i in range(rows):
            for j in range(cols):
                if(grid[i][j]=='1'):
                    num_islands+=1
                    dfs(i,j)
        return num_islands
```

【题意】
适用邻接矩阵存储图

2、 腐烂橘子，每分钟四个方向橘子腐烂
return: 没有新鲜橘子为止所必须经过的最小分钟数 | 如果不可能，返回 -1

【code】

【方法】
- 首先，适用于广度优先搜索
	- 逐层拓展
	- 每一层的腐败同时发生
- 腐败源不止一个
—— *多源广度优先搜索问题*

算法思路：值为2的橘子在网格中扩散的过程
1、初始化，获得腐烂源坐标
2、处理边界情况，若没有新鲜橘子，则不需要执行
3、开始BFS 模拟时间流逝
		逐层处理，需要记录每层的数量，使用queue管理
		处理完改成，若deque中还有元素，则说明有新感染的🍊，mintue+1
4、得出结果
		若 fresh_count 为0，则返回minutes
		若 fresh_count不为0，则说明存在感染不到的

【实现】
```python
class Solution:
    # ⭐️ 多源广度优先搜索
    def orangesRotting(self, grid: list[list[int]]) -> int:
        if not grid:
            return 0
        rows,cols = len(grid),len(grid[0])
        queue = collections.deque() # 用来管理腐烂橘子
        fresh_count = 0

        # 1 init 统计新鲜橘子数量 和 腐烂橘子坐标
        for r in range(rows):
            for c in range(cols):
                if grid[r][c]==1:
                    fresh_count+=1
                elif grid[r][c]==2:
                    queue.append([r,c])
        
        # 2 边界判断
        if fresh_count==0:
            return 0
        
        minutes = 0
        directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]

        # 3 BFS 📍多源广度优先搜索
        while queue:
            if fresh_count==0:
                break
            level_size = len(queue)

            for _ in range(level_size):
                r,c = queue.popleft()
                for dr,dc in directions: # 逐层检查
                    next_r,next_c = r+dr,c+dc

                    if 0<= next_r <rows and 0<= next_c <cols and grid[next_r][next_c]==1:
                        grid[next_r][next_c]=2
                        fresh_count-=1
                        queue.append([next_r,next_c])

            if queue:
                minutes+=1

        # 4 结果判断
        return minutes if fresh_count==0 else -1
```

- 若不存在环，为有向无环图，则可以完成

```python
class Solution:
    # BFS，模拟课程学习路径
    def canFinish(self, numCourses: int, prerequisites: list[list[int]]) -> bool:
        if numCourses<=1:
            return True
        
        # 1、图的表示和入度计算
        adj=[[] for _ in range(numCourses)] # 使用邻接表来表示，包含所有以i为先驱的节点
        in_degree=[0]*numCourses
        for course,prereq in prerequisites:
            adj[prereq].append(course)  # 邻接表，表示以当前节点为前缀的节点有哪些   # 图的表示方法，邻接表、邻接矩阵
            in_degree[course]+=1  # 入度，表示当前课程需要几个前缀
        
				# 2、初始化就绪列表
        queue = collections.deque() # 存储已就绪课程
        for i in range(numCourses):
            if in_degree[i]==0:
                queue.append(i)

        # 3、BFS过程，拿出课程，表示已完成
        course_taken = 0
        while queue:
            prereq_course = queue.popleft()
            course_taken+=1 # 完成课程+1
            
            # 已完成课程后，更细该课程后续课程
            for next_course in adj[prereq_course]:
                in_degree[next_course]-=1
                if in_degree[next_course]==0: # 入度为0表示就绪
                    queue.append(next_course)
                    
				# 4、判断最终结果
        return course_taken == numCourses
```

```python
        self.root = TrieNode() 
    def insert(self, word: str) -> None:
        '''
        插入单词
        '''
        curr = self.root
        for char in word:
            if char not in curr.children: # 检查的是所有子节点key
                curr.children[char] = TrieNode()
            curr = curr.children[char]
        curr.is_end_of_word=True
        

    def search(self, word: str) -> bool:
        '''
        搜索单词
        '''
        curr = self.root
        for char in word:
            if char not in curr.children:
                return False
            curr = curr.children[char]
        return curr.is_end_of_word

    def startsWith(self, prefix: str) -> bool:
        '''
        检查是否存在前缀
        '''
        curr = self.root
        for char in prefix:
            if char not in curr.children:
                return False
            curr= curr.children[char]
        return True
```

## 回溯
==回溯法（Backtracking）==是一种通过探索所有可能的候选解来找出所有解的算法。如果候选解被确认不是一个解（或者至少不是最后一个解），回溯算法会通过在上一步进行一些变化来丢弃该解，即“回溯”到上一个决策点，尝试其他的选择。
类比，*走迷宫*
1. 你站在一个路口，面前有多条路可以选择。
2. 你**选择一条路**往前走。
3. 你继续走，又遇到新的路口，再做选择...
4. 当你走到一个**死胡同**（此路不通，不满足题目条件），或者已经找到了一个出口（一个完整的解）时，你就停下来。
5. 然后，你**原路返回**到上一个路口，并**尝试你没走过的另一条路**。
6. 你重复这个过程，直到探索完所有从起点开始的路径。
原路返回到上一个路口，并尝试另一条路”的过程，就是**回溯**。

在代码中，这个过程通常通过**递归**来实现，其本质是一种“**暴力搜索的剪枝优化**”。它系统地搜索一个问题的所有解或最优解，但与纯粹的暴力搜索不同，当它发现当前选择的路径不可能得到正确解时，会立即停止，退回上一步，从而避免了大量无效的计算。

📍总而言之，回溯是一种强大通用的搜索算法，用于解决组合搜索问题。
### 回溯通用模板
三段式结构，通常递归实现
- 路径 (Path)：已经做出的选择的集合。
- 选择列表 (Choices)：当前可以做的选择的集合。
- 结束条件 (End Condition)：通常是路径的长度达到了题目要求，或者选择列表为空。

```python
backtrack(路径, 选择列表):
    if (满足结束条件):
        记录/保存结果
        return

    for 选择 in 选择列表:
        1. 做选择:
           将该选择从选择列表中移除
           将该选择添加到路径中

        2. 递归:
           backtrack(路径, 选择列表)  // 进入下一层决策

        3. 撤销选择 (回溯的关键！):
           将该选择从路径中移除
           将该选择重新加入选择列表
```

### 如何理解返回值为bool的递归函数?

❗ 需要特别注意返回值

1. 当最终返回Ture的时候，需要向上层返回true，然后逐层返回到最顶层
```python
if backtrack(next_step):
		return True
```
2. 当一次循环中返回false时，需要执行下次循环，对应下次
3. 当所有循环均返回false时，向上层返回false。当所有子函数均没有返回ture时，上层函数返回flase


#### 模板⭐⭐⭐
```python
def can_solve(arguments...):
    # 1. 基本情况 (Base Cases) - 递归的出口

    # 1a. 成功的出口：直接回答 "是"
    if (已经找到了明确的解):
        return True

    # 1b. 失败的出口：明确知道此路不通 (剪枝)
    if (当前状态已经非法或越界):
        return False

    # 2. 递归步骤 (Recursive Step) - 委托任务

    # 标记当前状态为“已访问”，避免重复
    mark_as_visited(current_state)

    # 遍历所有可能的“下一步”
    for next_step in all_possible_next_steps: # 2️⃣
        
        # 委托任务：问你的“手下”能否从下一步解决问题
        if can_solve(next_step_arguments...):
            # 如果手下说“能”，太好了！
            # 撤销标记（可选，但好习惯），然后立刻报告成功
            unmark_as_visited(current_state)
            return True   # 1️⃣
    
    # 循环结束，说明所有“下一步”都走不通
    # 撤销标记，为其他路径的探索恢复现场
    unmark_as_visited(current_state)
    
    # 向上级报告：从我这里出发，解决不了问题
    return False # 3️⃣ 当所有自函数均没有返回True时，上层函数返回False
```

```python
class Solution:
    def solveNQueens(self, n: int) -> list[list[str]]:
        '''
        适合用回溯
        定下前一个元素后，再考虑后续元素
        每个元素遍历每行所有的位置
        '''
        result=[]
        board = [['.' for _ in range(n)] for _ in range(n)]

        # 用于快速判断的集合 （若每次遍历矩阵效率低
        columns = set()
        diagonals1 = set() # 主对角线 r - c
        diagonals2 = set() # 副对角线 r + c


        def backtrack(row:int):
            # 终止条件
            if row == n:
                temp_solution = ["".join(row_list) for row_list in  board]
                result.append(temp_solution)
                return
            
            # 遍历当前所有列，放置 Q
            for col in range(n):
                # 剪枝
                if col in columns or (row-col) in diagonals1 or  (row+col) in diagonals2:
                    continue

                # 1
                board[row][col]='Q'
                columns.add(col)
                diagonals1.add(row-col)
                diagonals2.add(row+col)

                # 2
                backtrack(row+1)

                # 3
                board[row][col]='.'
                columns.remove(col)
                diagonals1.remove(row-col)
                diagonals2.remove(row+col)


        backtrack(0)
        return result
```

## 二分查找
*一、核心思想 (What is it?)*
二分查找（Binary Search）是一种在**有序数组**中查找特定元素的高效算法。
它的核心思想非常像“猜数字游戏”或“查字典”：
2. 你猜一个 1 到 100 之间的数字，我告诉你“大了”或“小了”。你不会从 1, 2, 3... 挨个猜，而是会先猜中间的 50。
3. 如果我告诉你“小了”，你会立刻排除 1-50，然后在 51-100 的中间（75）再猜。
4. 每一次猜测，你都能将“搜索范围”缩小一半。
这种“**分而治之**”（Divide and Conquer）的策略，就是二分查找的精髓。

*二、前提条件 (Prerequisite)*
二分查找**必须**应用于一个==有序==的数据集合（通常是数组）。如果数据是无序的，二分查找将无法正常工作。

*三、算法步骤 (How does it work?)*
我们使用三个指针来执行查找：`left`（左边界）、`right`（右边界）和 `mid`（中间位置）。

1. **初始化**：
    - `left` 指向数组的第一个元素，即 `left = 0`。
    - `right` 指向数组的最后一个元素，即 `right = len(array) - 1`。
2. **循环查找**：只要 `left <= right`（表示搜索区间还有效），就不断循环：
    - **计算中间位置**：`mid = left + (right - left) // 2`。
    - **比较**：将中间位置的元素 `array[mid]` 与目标值 `target` 进行比较：
        - **情况一：找到了！** 如果 `array[mid] == target`，查找成功，返回索引 `mid`。
        - **情况二：中间值太小**。如果 `array[mid] < target`，说明目标值只可能在 `mid` 的**右侧**。因此，我们抛弃左半部分，将搜索区间的左边界更新为 `left = mid + 1`。
        - **情况三：中间值太大**。如果 `array[mid] > target`，说明目标值只可能在 `mid` 的**左侧**。因此，我们抛弃右半部分，将搜索区间的右边界更新为 `right = mid - 1`。
3. **查找失败**：如果循环结束（此时 `left > right`），说明搜索区间已经为空，但仍未找到目标值。查找失败，通常返回 `-1`。

*四、代码模板*
二分查找主要有两种实现方式：循环（迭代）和递归。**循环是更常用、更推荐的方式**，因为它效率更高且没有栈溢出的风险。

#### 循环实现
```python
def binary_search(nums: list[int], target: int) -> int:
    """
    在一个有序数组中查找目标值的索引。
    如果找到，返回其索引；否则，返回 -1。
    """
    left, right = 0, len(nums) - 1

    # 循环条件：left <= right 确保搜索区间 [left, right] 有效
    while left <= right:
        # 防止 left + right 整数溢出的写法，在 Python 中非必需，但在 C++/Java 中是好习惯
        mid = left + (right - left) // 2

        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            # 目标在右侧，搜索区间变为 [mid + 1, right]
            left = mid + 1
        else: # nums[mid] > target
            # 目标在左侧，搜索区间变为 [left, mid - 1]
            right = mid - 1
            
    # 循环结束，未找到
    return -1
```

#### 递归实现
```python
def binary_search_recursive_util(nums, target, left, right):
    # 基本情况：搜索区间无效，未找到
    if left > right:
        return -1

    mid = left + (right - left) // 2

    if nums[mid] == target:
        return mid
    elif nums[mid] < target:
        # 递归搜索右半部分
        return binary_search_recursive_util(nums, target, mid + 1, right)
    else:
        # 递归搜索左半部分
        return binary_search_recursive_util(nums, target, left, mid - 1)

def binary_search_recursive(nums: list[int], target: int) -> int:
    return binary_search_recursive_util(nums, target, 0, len(nums) - 1)
```

*五、时间复杂度分析*
- **时间复杂度：O(log n)**  
    这是二分查找最大的优势。每次操作都将数据规模减半。对于一个大小为 `n` 的数组，最多需要 `log₂(n)` 次比较。例如，对于 100 万个元素，最多只需要约 20 次比较，而顺序查找最坏需要 100 万次。
- **空间复杂度：O(1)**  
    对于循环实现，我们只使用了固定的几个变量，不随数据规模 `n` 变化。  
    （对于递归实现，空间复杂度为 O(log n)，因为需要递归调用栈空间）。

*六、常见陷阱与变形*
1. **循环条件**：应该是 `while left <= right` 而不是 `while left < right`。使用 `<` 会导致当 `left == right`（搜索区间只剩一个元素）时循环提前退出，错过这个元素。
2. **边界更新**：应该是 `left = mid + 1` 和 `right = mid - 1`。因为 `mid` 已经被检查过了，所以新的搜索区间不应再包含它。
3. **变种问题**：二分查找的思想不仅能用于“查找特定值”，还能解决更广泛的问题，例如：
    - **寻找第一个/最后一个等于 `target` 的元素**。
    - **寻找第一个大于/小于 `target` 的元素**。
    - **在一个旋转排序数组中查找元素** (如 LeetCode 33)。
    - **答案在某个范围内的求解问题**：将问题的“答案”作为搜索对象，通过二分法来“猜答案”，并用一个检查函数 `check(mid)` 判断当前猜的答案是偏大还是偏小。
这些变种问题的关键在于**如何根据 `check(mid)` 的结果来调整 `left` 和 `right` 的边界**，以确保解始终在搜索区间 `[left, right]` 内。

【方法】
**思路**
1. 问题本质：寻找一个==分割线==，使得num1和num2 左侧的元素均小于等于右侧的元素
	1. 总的左侧与右侧元素数相等。因此，在num1中确定分割线后，num2中直接会有对应分割线
2. 建立二分查找模型
	1. num1，num2分别有m,n个元素
	2. 在i处分割，i一定为整数
		1. num1，左侧 `nums1[0], ..., nums1[i-1]` (共 i 个元素)；右边部分：`nums1[i], ..., nums1[m-1]` （m-i个元素）
		2. 为了满足分割线要求，num2会存在对应位置的割线 j ` i + j = (m + n + 1) // 2`
			1. m+n为偶数，则i+j=(m+n)/2，中位数需要是左右的平均值
			2. m+n为奇数，则i+j比剩余部分大一，中位数为左侧最大值
3. 正确分割条件` max(L1, L2) <= min(R1, R2)` ==正确条件==
	1. 简化，等价为 （因为原本）
		1.  `nums1[i-1] <= nums2[j]`
		2.  `nums2[j-1] <= nums1[i]`
4. 二分查找调整策略
	1. i偏小,需要右移，表现 `nums2[j-1] > nums1[i]`
	2. i偏大，需要左移，表现 `nums1[i-1] > nums2[j]`
5. 正确结果
	1. 与二分查找模型对应 (左侧数目等于右侧 或左侧大一)
	2. 偶数时， `(max(nums1[i-1], nums2[j-1]) + min(nums1[i], nums2[j])) / 2`
	3. 奇数时，`max(nums1[i-1], nums2[j-1])`

```python
class Solution:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        if len(nums1) > len(nums2): # 优化，使得二分查找在较短数组上执行
            nums1,nums2=nums2,nums1
        m,n=len(nums1),len(nums2)
				
        total_left =(m+n+1)//2 # 1️⃣

        left,right=0,m

        while left<=right:
            i=(left+right)//2 # 调整i，寻找分界点
            j=total_left - i

            # 处理四个关键值 注意索引边界值 2️⃣
            # 如果 i=0, nums1_left_max 不存在，设为负无穷
            nums1_left_max = float('-inf') if i == 0 else nums1[i - 1]
            # 如果 i=m, nums1_right_min 不存在，设为正无穷
            nums1_right_min = float('inf') if i == m else nums1[i]
            # 同理处理 nums2
            nums2_left_max = float('-inf') if j == 0 else nums2[j - 1]
            nums2_right_min = float('inf') if j == n else nums2[j]
            
            # 四个关键值判断
            if nums1_left_max<=nums2_right_min and nums2_left_max<=nums1_right_min:
                # 成功条件
                left_max=max(nums1_left_max,nums2_left_max)
                if (m+n)%2==1:
                    return left_max # 总长度为奇数
                else:
                    right_min=min(nums1_right_min,nums2_right_min)
                    return (left_max+right_min)/2.0  # 总长度为偶数
            elif nums1_left_max>nums2_right_min: # i 需要左移
                right=i-1
            else: #i需要右移 nums2_left_max >nums1_right_min
                left=i+1
            
        return -1
```

	- 从而不影响 `min(nums1_right_min, nums2_right_min)` 的结果
- `j=0` 和 `j=n` 的情况同理

## 栈
**Stack**
栈是一种基础且非常重要的数据结构，它的核心特点是 **后进先出 (Last-In,First-Out, LIFO)**。
你可以把它想象成一摞盘子，你总是先放盘子在最上面，也总是从最上面取走盘子。

#### 栈的主要知识点
包括：
*一、基本概念与操作*
- **定义**：一种只允许在一端（称为**栈顶**，Top）进行插入和删除操作的线性表。
- **核心操作**：
    - `push`：**入栈/压栈**。在栈顶添加一个新元素。
    - `pop`：**出栈/弹栈**。移除并返回栈顶的元素。
    - `peek` 或 `top`：**查看栈顶元素**。返回栈顶元素，但不移除它。
    - `isEmpty`：**判空**。检查栈是否为空。
    - `size`：**获取大小**。返回栈中元素的数量。
- **时间复杂度**：对于标准实现，以上所有核心操作的时间复杂度都是 **O(1)**。

*二、实现方式*
栈可以通过其他数据结构来实现：
1. **使用数组/列表 (Array/List)**：这是最常见的方式。Python 中的 `list` 类型就可以很方便地模拟栈，`list.append()` 对应 `push`，`list.pop()` 对应 `pop`。
2. **使用链表 (Linked List)**：将链表的头部作为栈顶，入栈和出栈操作都只在头部进行，时间复杂度也是 O(1)。

*三、典型应用场景*
栈的 LIFO 特性使其在处理具有“后进先出”逻辑的问题时非常有用：
1. **括号匹配**：检查一个字符串中的括号（如 `()`, `[]`, `{}`）是否成对闭合。这是最经典的栈应用之一（例如 LeetCode 第20题）。
2. **函数调用栈 (Call Stack)**：程序语言在内部使用栈来管理函数调用。每当一个函数被调用，它的信息（返回地址、参数、局部变量）就会被 `push` 到调用栈上；当函数执行完毕返回时，它的信息会被 `pop` 出栈。**递归**的实现就深度依赖于函数调用栈。
3. **表达式求值/转换**：
    - **后缀表达式（逆波兰表达式）求值**：例如计算 `2 3 + 5 *`。
    - **中缀表达式转后缀表达式**：例如将 `(2 + 3) * 5` 转换为 `2 3 + 5 *`。
4. **深度优先搜索 (DFS)**：在图和树的遍历中，DFS 的迭代（非递归）版本通常使用栈来实现。
5. **浏览器的“后退”功能**：每访问一个新页面，就将其 `push` 到栈中。点击“后退”时，就从栈中 `pop` 出上一个页面。
6. **文本编辑器的“撤销”(Undo)功能**：每执行一个操作，就将该操作 `push` 到栈中。点击“撤销”时，就 `pop` 出最近的操作并执行其逆操作。

#### 题目特点
具备以下一个或多个特点：
1. **配对与消除 (Pairing and Elimination)**：
    - **特点**：问题中存在成对出现的元素，当一个“右”元素出现时，需要与最近一个未被匹配的“左”元素进行配对和消除。
    - **关键词**：括号匹配、符号抵消、成对出现。
    - **例子**：有效的括号、简化文件路径、消消乐游戏。
2. **顺序依赖与逆序处理 (Sequence Dependency & Reverse Order)**：
    - **特点**：问题求解过程中，当前元素的决策或结果，依赖于它之前（或之后）的、且离它最近的某个元素。栈可以用来“暂存”这些元素，直到找到能处理它们的那个元素。
    - **关键词**：下一个更大的元素、最近的、左侧/右侧第一个...
    - **这种模式通常会引出一种特殊的栈用法，叫做“单调栈”(Monotonic Stack)。**
3. **递归与回溯的模拟 (Simulating Recursion & Backtracking)**：
    - **特点**：问题的解决路径呈现出深度优先的探索模式，就像函数调用一样，进入更深层次，然后返回。
    - **关键词**：深度优先搜索 (DFS)、树的遍历（前序、中序、后序的迭代法实现）。
    - **解释**：操作系统本身就是用“函数调用栈”来管理递归的。因此，任何递归问题理论上都可以用一个手写的栈来改成迭代形式，以避免递归深度过大导致的栈溢出。

最常见的两类问题
##### 配对消除

*核心思想*
这类问题的本质是处理成对出现的元素，当一个“结束”符号出现时，需要与离它**最近**的一个“开始”符号进行匹配和抵消。

*问题特征*
- 问题中存在**成对**的、需要**匹配**的元素。
- 匹配规则具有**就近原则**，即一个结束符号总是匹配最近的那个未被匹配的开始符号。
- 常常涉及**合法性校验**（如括号是否合法）或**化简**（如消除冗余部分）。

*通用解题模板*
1. 初始化，栈、匹配关系哈希表
2. 遇到开始符号，入栈
3. 遇到结束符号，判断是否匹配
```python
def pairing_elimination_template(sequence):
    stack = []
    # mapping = { 'end_symbol': 'start_symbol' } # 可选的配对关系

    for element in sequence:
        if is_start_symbol(element):
            # 1. 遇到“开始”符号，一律入栈
            stack.append(element)
        elif is_end_symbol(element):
            # 2. 遇到“结束”符号，进行判断
            # a. 如果栈为空，说明没有匹配的开始符号，无效
            if not stack:
                return False # or handle as invalid
            
            # b. 弹出栈顶的“开始”符号
            start_symbol = stack.pop()

            # c. 检查是否匹配
            if not is_match(start_symbol, element):
                return False # or handle as invalid
    
    # 3. 遍历结束后，如果栈为空，说明所有符号都完美配对
    return not stack
```

- 739，每日温度

*核心思想*
单调栈是一种特殊的栈，它在操作过程中始终保持栈内元素的**单调性**（单调递增或单调递减）。它的主要作用是高效地找到一个元素**左边或右边第一个比它大/小**的元素。

*核心机制*
当一个新元素将要入栈时，为了维持栈的单调性，我们会不断地将栈顶不符合单调性的元素弹出。**而这个弹出的时刻，就是我们找到答案的时刻！**
- 对于被弹出的元素 `stack.top()` 来说，新来的这个元素 `new_element` 就是它右边第一个破坏单调性（即第一个比它大/小）的元素。

*问题特征*
- 题目要求寻找每个元素**左/右侧**的**第一个**更大/更小的元素。
- 关键词：“下一个更大”、“上一个更小”、“最近的大于”等。
- 问题可以转化为求解每个元素与其左/右侧第一个关键元素之间的**距离**或**关系**。

*通用解法模板 (以“寻找右侧第一个更大”为例，递减栈)*
```python
def monotonic_stack_template(nums):
    n = len(nums)
    result = [-1] * n # 初始化结果数组
    stack = [] # 栈中存储的是元素的索引

    for i in range(n):
        # 1. 核心：当栈不为空，且当前元素大于栈顶索引对应的元素时
        while stack and nums[i] > nums[stack[-1]]:
            # 2. 找到了栈顶元素的“右侧第一个更大元素”
            top_index = stack.pop()
            
            # 3. 记录结果（可以是值，也可以是距离）
            result[top_index] = nums[i] # 或者 result[top_index] = i - top_index
        
        # 4. 将当前元素的索引入栈，维持单调性
        stack.append(i)
        
    return result
```

### 20 有效的括号_low
【题意】
给定一个只包括 `'('，')'，'{'，'}'，'['，']'` 的字符串 s ，判断字符串是否有效。
1、左括号必须用相同类型的右括号闭合。
2、左括号必须以正确的顺序闭合。
3、每个右括号都有一个对应的相同类型的左括号。

【code】


【方法】
1. 问题本质，*一个右括号总是与它左边最近的、未被匹配的那个左括号进行配对*
	1. 适合使用栈
2. 使用==列表==模拟栈、使用==哈希表(字典==)记录配对关系
	1. 键为，右括号
	2. 值为，左括号
3. 具体执行
	1. 遍历字符串
	2. 若遇到左括号，入栈
	3. 若遇到右括号，判断栈顶元素是否匹配
	4. 若遍历整个操作完成，栈为空，则说明字符串合法

【实现】
```python
class Solution:
    def isValid(self, s: str) -> bool:
        if len(s)%2==1: # 若长度为奇数则直接不用判断
            return False
        
        # 在python中使用list模拟栈
        stack = []
        # 配对关系的哈希表
        pairs = {
            ")": "(",
            "]": "[",
            "}": "{",
        }

        # 📍 遍历每个元素进行判断
        for char in s:
            if char in pairs: # 右括号
                if not stack or stack.pop()!=pairs[char]:
                    return False
            else:   # 左括号
                stack.append(char)
                    
        return not stack
```

### 155 最小栈
【题意】
设计一个支持 push ，pop ，top 操作，并能*在常数时间内检索到最小元素的栈。*
【code】

【方法】
- 难点，常数时间检测到最小值
	- 想到，拿空间换时间
- 解决
	- 在每次操作时就维护好当前的最小值
	- 添加额外的辅助栈`min_stack`，栈顶保存当前`self.stack`中的最小值
	- 随着stack，pop、push而更新

【实现】
```python
class MinStack:
    def __init__(self):
        """
        使用一个主栈 stack 和一个辅助栈 min_stack
        - stack: 正常存储所有元素
        - min_stack: 栈顶始终保存当前 stack 中的最小值
        """
        self.stack=[]
        self.min_stack = []
    
    def push(self, val: int) -> None:
        self.stack.append(val)
        if not self.min_stack or val<=self.min_stack[-1]:
            # 只有当 min_stack 为空，或新元素小于等于当前最小值
            # 等于，情况也需要加
            self.min_stack.append(val)
        
    def pop(self) -> None:
        popped_val = self.stack.pop()
        if popped_val==self.min_stack[-1]:
            self.min_stack.pop()

    def top(self) -> int:
        return self.stack[-1]
        
    def getMin(self) -> int:
        return  self.min_stack[-1]
```

### 394 字符串解码
【题意】
给定一个经过编码的字符串，返回它解码后的字符串。
编码规则为: `k[encoded_string]`，表示其中方括号内部的 encoded_string 正好重复 k 次。注意 k 保证为正整数。
输入字符串总是有效的；输入字符串中没有额外的空格，且输入的方括号总是符合格式要求的
- 原始数据不包含数字
- 数字只表示重复次数

【code】

【方法】
本质，栈能完美处理“嵌套”具有“先进后出”的结构
1. 将`[`看作进入一个更深层次的开始
2. 把`]`看作当前层次的结束，并返回上一层
3. 当进入新层次时，保存上一层状态
4. 返回上一层的时候，将保存的状态恢复，并加上新层次的东西

【实现】
```python
class Solution:
    def decodeString(self, s: str) -> str:
        '''
        将[看作进入一个更深层次的开始
        把]看作当前层次的结束，并返回上一层
        当进入新层次时，保存上一层状态
        返回上一层的时候，将保存的状态恢复，并加上新层次的东西
        '''
        stack= []
        res= '' # 记录当前层解码的字符串
        multi=0
        for char in s:
            if '0'<= char <='9':
                multi = multi*10 + int(char) # 1️⃣ 处理数字
            elif char=='[':
                # 当前倍数，上一层字符串结果，入栈
                stack.append((multi,res))
                multi=0
                res=''
            elif char==']': #解码并更新res
                last_multi,last_res = stack.pop()
                res=last_res+last_multi*res # 执行解码，更新res
            else:
                res+=char # 记录当前层字符
        return res
```

### 739 每日温度
【题意】
每日温度 
temperatures ，表示每天的温度
返回一个数组 answer ，其中 `answer[i]` 是指对于第 i 天，下一个更高温度出现在几天后

【code】

【方法】
**问题关键**，对于一个温度，寻找右侧第一个比他大的数

思路：==单调栈==
		这个栈存储的是日期的索引
		从栈底到栈顶，存储的温度是单调递减的

1. 对每天温度入栈
2. 当新的温度，比栈顶高，则说明找到了更高的那天    
		1. 弹出栈顶索引，`当前索引-栈顶索引`，即为天数差
		2. 继续与新的栈顶对比，直到，栈为空或当前温度不比栈顶高
		3. 将今天温度入栈
3. 最后栈中可能还剩余元素，他们不存在比起很高的温度，因此结果是0


【实现】
```python
class Solution:
    def dailyTemperatures(self, temperatures: list[int]) -> list[int]:
        n=len(temperatures)
        answer = [0]*n
        stack=[]
        for i,temp in enumerate(temperatures):
            # 核心逻辑
            # 当栈不为空，当前温度大于栈顶元素，则找到结果
            while stack and temp>temperatures[stack[-1]]:
                prev_index = stack.pop()
                answer[prev_index]=i- prev_index # 计算天数差
            stack.append(i)
        return answer
```

### 84 柱状图中的最大矩阵_high
【题意】
柱状图中最大的矩阵
给定 n 个非负整数，用来表示柱状图中各个柱子的高度。每个柱子彼此相邻，且宽度为 1 。
求在该柱状图中，能够*勾勒出来的矩形的最大面积*。

【code】


【方法】
本质上，面积=`h*w`，寻找当前柱子，能向左，向右延展的最大宽度
——由左右第一个比它矮的柱子决定

==单调栈==，寻找左/右第一个更小/更大的元素”正是**单调栈**的经典应用场景。我们可以通过一次遍历就计算出所有这些边界。

左边界- 由栈内小于当前元素的坐标决定（并不包含该值）
右边界- 由当前遍历的元素决定

- 对于柱子 `i`，它的高度是 `h = heights[i]`。
- 我们需要找到它左边第一个比 `h` 矮的柱子的索引 `left_boundary`。 不包含
- 我们需要找到它右边第一个比 `h` 矮的柱子的索引 `right_boundary`。右边界即为本身
- 那么，以柱子 `i` 为高的矩形，其宽度就是 `right_boundary - left_boundary - 1`。
- 面积就是 `heights[i] * (right_boundary - left_boundary - 1)`。

【实现】
```python
class Solution:
    def largestRectangleArea(self, heights: list[int]) -> int:
        heights=[0]+heights+[0]# 简化边界处理
        n = len(heights)
        stack = [] # 单调栈，从小到大
        max_area = 0
        for i in range(n):
            # 核心逻辑，当新的元素小于栈顶元素时，出栈计算面积
            # 💡此时面积不会稳定增大（当下一个元素大于栈顶元素时，面积为稳定增大）
            while stack and heights[i]<heights[stack[-1]]: #寻找面积的左右边界
                top_index = stack.pop()
                h = heights[top_index]
                left_boundary_index = stack[-1]
                w=i - left_boundary_index -1 # 左边界这一列并不算进来
                max_area=max(max_area,h*w)
            
            # 单调栈，每个元素均入栈
            stack.append(i) # 值为
        return max_area
```

### for & range
> 总结，range不管正序，倒序，均不包含最后`stop`那个数组
> - 正序，最后数字为列表长度
> - 倒序，最后数字为-1

`range()` 函数主要用于生成一个**整数序列**，它通常与 `for` 循环一起使用，用来指定循环执行的次数。

三种形式

| **形式**                         | **描述**                                         | **示例**                                       |
| ------------------------------ | ---------------------------------------------- | -------------------------------------------- |
| **`range(stop)`**              | 生成从 **0** 开始，到 `stop-1` 结束的整数序列。               | `range(5)` 会生成 `0, 1, 2, 3, 4`               |
| **`range(start, stop)`**       | 生成从 `start` 开始，到 `stop-1` 结束的整数序列。             | `range(2, 7)` 会生成 `2, 3, 4, 5, 6`            |
| **`range(start, stop, step)`** | 生成从 `start` 开始，到 `stop-1` 结束，步长为 `step` 的整数序列。 | `range(1, 10, 2)` 会生成 `1, 3, 5, 7, 9`        |
| 倒序遍历                           | `stop`，必须是序列中最小值减去步长的绝对值                       | `range(list_length - 1, -1, -1)` 倒序遍历整个 list |

举例（🟥易错）：
- **`range(i,1,-1)`**
	- 从 i 开始，递减到 1，且不包括 1
	- 即，i，i-1，...，2
- **`range(i,-1,-1)`**
	- 从 i 开始，递减到 -1（不包含-1）

### 面向对象编程-OOP
`self` 代表**类的实例本身**

#### self.变量
`self.变量`（例如 `self.stack`）就是存储在这个对象“名片”上的信息。这种变量被称为**实例变量 (Instance Variable)** 或**实例属性 (Instance Attribute)**。

特点：
1. 属于对象个体，每个实例对象有自己独立的一套实例变量
2. 在类的所有方法中共享，类的其他任何方法都可以通过`self`来访问修改
3. 生命周期与对象一致

使用 `self.变量` 的根本原因是为了**保存对象的状态 (State)**。一个对象不仅仅包含行为（方法），还需要有自己的数据（状态）。如果没有 `self.变量`，对象就无法“记住”任何信息。

## 典型报错❌



#### 变量作用域问题
```python
class Solution:
    def diameterOfBinaryTree(self, root: TreeNode) -> int:
        ans=0  # ⭐
        def depth(node:TreeNode)->int:
            if not node:
                return 0
            L = depth(node.left)
            R = depth(node.right)
            if L+R > ans: # ❗报错位置
                ans=L+R
            return max(L,R) + 1
        
        depth(root)
        return ans
```
❌ `UnboundLocalError`: cannot access local variable 'ans' where it is not associated with a value

【原因分析】
- 这个错误是因为在Python中，当你在一个嵌套函数内部试图修改外层函数的变量时，如果该变量是*不可变类型（如int、str等）*，Python会认为你在内层函数中创建了一个新的局部变量，而不是使用外层函数的变量。这就是所谓的"变量作用域"问题。

【解决】
- 在嵌套函数内部声明 `nonlocal ans`
- 或使用 `self.ans`
	- 此时访问的是类的实例对象的属性，而非局部变量
- 可变容器 
	- 如list `ans=[0]`，后续修改使用角标访问
